# `cli`

### `GameLoop`
* Ответственен за всю игровую логику (переходы между состояниями)

# `cli.io`

### `IOProcessor`
* Ждет ввода команды (в отдельном потоке)
* Парсит команду и ее аргументы
* Валидирует ввод и вывод сообщение об ошибке
* Получает `Executable` с помощью `CommandFactory`
* Исполняет команду; если команда игровая, отправляет в общую очередь результат исполнения

### `commands.CommandFactory`
* По константе `InputCommand` создает экземпляр команды `Command` с аргументами

### `commands.Executable`
* Интерфейс команд
* Имеет метод `CommandResult<D> execute()`

### `commands.Command`
* Абстрактный класс команд
* Реализует `Executable`
* Имеет метод `D convertArgs(String[] args)`

### Реализации `commands.Command`
* Инкапсулируют логику конвертации аргументов в DTO (`GameDTO`, `MoveDTO`) или выполнения команды
